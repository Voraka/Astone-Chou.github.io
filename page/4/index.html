Lua 源码分析--执行流程

简介

　　与大多数的脚本语言一样, lua 的执行流程也分为编译和运行两个部分. lua 首先将 lua 代码编译成字节码, 然后 lua 虚拟机按照字节码指令执行 lua. 编译部分又分为词法分析、语法分析、指令生成. 运行部分主要由 lua 虚拟机按照生成的 OPCODE 执行 lua. 在 lua 源码分析–工具中介绍了简化版 lua 解释器的工程, 可以通过这个工程来分析 lua 的执行流程.

编译

　　从 main 函数开始, lua 调用 luaL_loadfile 加载 main.lua, 将 main.lua 文件内容载入内存作为一个 chunk. 然后, 调用 luaD_protectedparser 开始进编译. luaD_pcall 和 luaD_rawrunprotected 用于进行保护性调用, 在出错的时候使用 longjump 跳出. luaD_pcall 的第二个参数 func 就是真正用于解析的函数 f_parser. f_parser 涉及到词法分析、语法分析等编译原理的内容就不深入分析了. 最终经过 f_parser 编译之后将 main.lua 作为一个 LClosure 放置在栈顶. lua 编译流程的 CallStack 如下:

Lua 数据结构--State

State

　　lua 的 State 主要分成两个部分, 一个是线程 State, 一个是虚拟机 State. 线程 State 的结构是 lua_State. 虚拟机 State 的结构是 global_State. 注意, 这里的线程是指 lua 的线程, 而不是操作系统意义上的线程. 一个 lua 虚拟机只有一个 global_State, 多个 lua_State 被 global_State 管理.

lua_State

　　lua_State 是 lua 中非常重要的数据结构. 一个 lua_State 就是一个独立的运行栈, 包含了与 C 交互的栈信息, lua 函数调用信息, 错误处理, 虚拟机行为 hook 等等.

Lua 数据结构--Udata

Udata

　　在 lua 中 userdata 有两种实现，LUA_TLIGHTUSERDATA 和 LUA_TUSERDATA。light userdata 使用一个 void* 指针实现，full userdata 使用 Udata 结构实现。userdata 用于 C 和 lua 之间进行结构数据交互。将 C 提供的方法导入 lua 中，可以很方便的操作 C 提供的结构。而 lua 只能通过提供的方法来操作 userdata。这种方式类似于，用 C/C++ 写好了积木，由 lua 将这些积木拼接成想要的样子，这是 lua 最常见的使用场景。
