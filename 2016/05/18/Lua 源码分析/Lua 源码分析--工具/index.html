Lua 源码分析

工具

　　作为一门脚本语言, lua 的源代码仅仅只有 23k 行代码是非常好的学习资料. 进行源码阅读通过一些辅助工具能够帮助阅读和理解代码. 下面介绍一些我用于 lua 源码分析的一些工具.

Source Insight

　　Source Insight 是非常好用的源码阅读工具. 支持如 C、C++、ASM、PAS、ASP、HTML 等常见语言, 还支持自定义关键字. 辅助窗口能显示 symbol, reference, callinfo, context. 最好用的莫过于方便的函数跳转, 同时还带有支持前进后退的跳转栈. 使用快捷键 [Ctrl + 左键] 进行符号跳转, [Alt + <] 后退, [Alt + >] 前进, [Shift + F8] 高亮.

GDB

　　分析源码另一个重要的部分就是调试了. 通过 gdb 调试能够方便的看到当前的调用栈, 而不需要频繁查看调用时的参数. 同时, 顺着调试能够很快梳理出函数的调用流程. 进行调试时对源码工程进行一些修改能够更方便的调试代码.
　　首先, 我们下载解压最新版本的 lua 源代码.

　　然后, 修改 lua 源码中的 Makefile. 为 lua 加上 -gdwarf-2 -g3 选项用于 gdb 调试, 将 lua 编译成一个 .a, 作为库连接在自己的程序中. 这样做的好处是在调试和增加新的 lib 时非常的方便.
